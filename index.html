AngularJS 四大特性

(1)MVC模型

Model：即业务数据 $scope.xx $rootScope.xx
View：即业务数据的呈现 HTML + ngXxx
Controller：负责操作业务数据 .controller(‘’, function(){})

(2)双向数据绑定

方向1：Model绑定到View，只要Model变View随着变
方向2：View(表单控件)绑定到Model，只要View变Model随着变

(3)依赖注入

如何解决依赖关系：

(1)主动创建方式
var c1 = new Car(); //主动创建依赖对象
var d1 = new Driver( c1 ); //传递依赖对象
(2)被动注入(Injection)方式
module.controller(‘控制器’, function($scope, $interval){})

可以被注入的对象——所有的service/provider对象都可以被注入

(1)$rootScope：在多个控制器间共享数据的服务
(2)$interval：提供周期性定时器服务
(3)$timeout：
(4)$log：提供五个基本的日志输出服务
(5)$http：提供异步HTTP请求（AJAX）服务
用法： $http({method:’GET’, url:’/xx.php’}).
success(fn).
error(fn);
简化版： $http.get(‘url’).success(fn);
$http.post(‘url’, data).success(fn);

(5)AngularJS提供的模块——ngRoute

Route：路由，通过某条线路找到目标内容。
ngRoute模块的用途：就是根据浏览器中URL中的一个特殊的地址标记(形如#/xxx)，查找到该标记所对应的模板页面，并异步加载到当前页面的ngView指令中。使用步骤：

(2)创建多个模板页面(习惯上放在一个特别的目录下，如tpl)
(3)创建Module，声明依赖于ng和ngRoute两个模块。
(4)在Module中配置路由字典。

webpack

  1. webpack即可以识别AMD commonJs ES6模块的工具，又可以进行打包同时可以建立自己的工作流（module bundler模块打包 工具）可以用于模块的打包、压缩、混淆。
    2.用于javascript模块管理的两大规范：
    1>CommonJs模块的同步加载，用于Node.js，
    2>AMD异步加载，通过requirejs用于前端。
    3.静态资源：Javascript、css、图片、Jade等。
    4.webpack的特性：
    1>丰富的插件–优化压缩代码
    2>有强劲的性能–开发时打开浏览器和加载文件速度加快
    3>loaders–将复杂依赖关系的js或者一些图片、css、编译打包成你即将部署到公网的一些文件
    4>支持–AMD、Commonjs
    5>减少初始化的加载时间
    6>优化
    7>Development Tools:支持http服务器、sourceMap
    8>多任务、多目标
    5.有独立配置的文件webpack.config.js
    6.webpack使用一步IO，并且具有多级缓存
    7.安装webpack与运行：
    1>创建工程目录结构：
    01.在index.html中写入：

    <script  src="app.js"></script>

    2>引入webpack依赖

    01.在工程中打开命令窗口输入命令：npm install -g webpack（下载全局的webpack）;
02.输入命令：npm install webpack --save-dev（下载本地webpack）;
03.输入npm info webpack（查看webpack的版本信息）；

    3>在工程中创建以下目录：

    01.webpack.base.config.js;
02.webpack.develop.config.js;
03.webpack.publish.config.js;

    4>在web.develop.config.js中配置：

    var path = require ('path' );

                                path : path. resolve( __dirname, 'develop/js'),

    5>在命令窗口输入webpack –config web.develop.config.js
    6>在app.js中写入：document.getelementById（”haha”).innerHTML = “有点难度!!!”;
    7>在package.json上写入：webpack –config web.develop.config.js
    9>为发布目录启动服务：

    01.在命令窗口输入：

    npm i webpack-dev-server –save
    （安装webpack-dev- server）

           (手动的在publish中添加index.html，并且把src="app.js"改为src="bundle.js")

git远程提交

步骤

  1. 建一个项目，在项目中打开命令窗口，输入git init对git进行初始化
  2. 在项目中添加文件夹（如：css js 并在文件夹中添加内容main.css main.js,）输入git status查看git的状态，查看文件夹是否添加，如果没有添加，输入git add . 进行添加
  3. 在命令窗口输入git commit -m “任意内容 “（这是进行日志提交）
  4. 将本地文件上传到远程仓库中（自己创建的githhub的仓库）在窗口中输入git remote add origin +github仓库的地址
  5. 在窗口中输入remote -v
  6. 在窗口中输入git push -u origin master（将本地文件推送到远程仓库中）

创建忽略清单文件

  1. 在窗口中输入（echo “ “ >.gitignore）(在本地项目中会自动生成gitignore文件，并在。gitignore）
  2. 输入git status查看git的状态
  3. 如果没有则添加输入git add .
